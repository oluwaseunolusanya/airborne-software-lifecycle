Introduction

Software lifecycle processes are a set of activities defined on a project to develop software product(s).

The sequencing of these activities and the assignment of responsibilities for them vary from project to project or industry to industry. These processes typical include planning, development, testing, deployment and maintenance.

The focus of this documentation will be the processes used in development software used in airborne systems and equipment like the engine control system on an aircraft. These are:

  • Software Planning Process
  • Software Development Process
  • Software Verification Process
  • Software Configuration Management Process
  • Software Quality Assurance Process
  • Certification Liaison Process

Software Planning

The purpose of the software planning process is to define the means of producing software that will satisfy its requirements and provide the level of confidence that is consistent with the software level. Software level in this context is the criticality of the system failure condition resulting from abnormal behaviour of software.

This process produces the software plans and standards that direct the software development processes and the integral processes.

Software Development

Software development processes produce one or more levels of software requirements. High-level requirements are produced directly through analysis of system requirements and system architecture. Usually, these high-level requirements are further developed during the software design process, thus producing one or more successive, lower levels of requirements.

NOTE: It may be required to justify software development processes that produce a single level of requirements
